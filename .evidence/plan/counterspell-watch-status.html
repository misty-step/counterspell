Ship Counterspell as an opt-in session watcher.

The work is done when counterspell status enumerates recent Claude transcript sessions across project dirs, maps them to Herdr panes by cwd, shows unmapped and live pane-only rows, and counterspell watch only plans compact-then-switch actions for explicitly configured targets.

Chosen Design

Rust binary, transcript discovery, TOML targets, Herdr as an external command. Claude JSONLs are the session source; Herdr remains the live pane source of truth; config is the only authority for remediation.

Standards

Strict opt-in defaults, path-normalized cwd matching, no hidden repo state, human-readable status output.

Acceptance

Tests cover drift detection, unattended gating, compact-then-switch sequencing, config parsing, discovery, unmapped sessions, live pane-only rows, ignored-without-target behavior, and persisted debounce state.

Verification

Repo gate, local install, live counterspell status, live counterspell watch, and a fresh diff critic.

Stop If

Herdr JSON shape contradicts the addendum, transcript parsing cannot recover session ids, or any untargeted session can arm actions.

Implementation

  • Document strict opt-in config and command behavior.
  • Add unit tests for transcript drift, gate blockers, remediation sequencing, config parsing, and opt-in arming.
  • Add CLI tests for all-project discovery, cwd mapping, unmapped sessions, live pane-only rows, and ignored drift.
  • Install locally and run the real commands against current Herdr panes.

Quality System

  • Gate: cargo fmt -- --check && cargo test && cargo clippy --all-targets -- -D warnings.
  • Fresh critic must check strict opt-in behavior, discovery completeness, status clarity, and test coverage against the lane card.
  • Residual risk to report: live session inventory is time-sensitive and reflects the machine state at verification time.
